## Description

## Problem

When the plugin's MCP server is installed (npm install, Claude Code plugin install), npm resolves transitive deps fresh from registry semver ranges. Even with AISDLC-439's signed top-level tarball SHA (DEC-0001), a compromised transitive dep ships with no integrity check — the `event-stream` 2018 incident shape.

DEC-0002 (Decision Catalog, resolved 2026-05-26): ship `npm-shrinkwrap.json` (the publishable lockfile npm honors for downstream installs) so every transitive dep's SHA-512 is pinned at plugin release time.

## Scope

- **Generate shrinkwrap**: run `npm shrinkwrap` in `ai-sdlc-plugin/mcp-server/`; commit the resulting `npm-shrinkwrap.json` to the repo.
- **Release-workflow regen**: extend `.github/workflows/release.yml` to regenerate `npm-shrinkwrap.json` before `pnpm publish` so each plugin release ships a fresh, valid shrinkwrap matching `package.json`.
- **CI sync gate**: hermetic test that fails CI if `npm-shrinkwrap.json` drifts from `package.json` (e.g. `package.json` updated but shrinkwrap not regenerated). Pattern: `npm install --dry-run --package-lock-only` vs committed shrinkwrap.
- **Operator runbook**: document the transitive-bump → shrinkwrap-regen flow. When Renovate / Dependabot opens a transitive bump PR, the workflow should auto-regenerate shrinkwrap (or the operator regenerates manually before merging).

## Non-goals

- Shipping shrinkwrap for `@ai-sdlc/pipeline-cli` or other published packages (only `@ai-sdlc/mcp-server` in this task; sibling packages follow up if/when threat model warrants).
- Replacing pnpm workspace lockfile at repo root (this is specifically for the publishable `mcp-server` package's transitive surface).

## DEC-0002 resolution rationale

Picked ship-shrinkwrap (Option A) over:
- **Option B (npm audit signatures)**: registry signature coverage is uneven across transitive deps; partial mitigation only.
- **Option C (pin direct deps, semver transitives)**: doesn't defend against transitive compromise; worst-of-both.
- **Option D (status quo)**: no install-time integrity guarantee for any dep is unacceptable given MCP server blast radius.

Patch-velocity cost (shrinkwrap blocks transitive patch updates until plugin re-release) mitigated by existing Dependabot/Renovate auto-PR flow that already drives transitive bumps via PRs.

Composes with DEC-0001 (AISDLC-439 — signed tarball SHA) for defense-in-depth.
