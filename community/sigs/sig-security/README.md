# SIG Security

The Security Special Interest Group reviews security aspects across all specification documents, the autonomy system, and policy enforcement.

## Scope

- Security implications of specification changes
- Autonomy level definitions and guardrails (`spec/autonomy.md`)
- Policy enforcement model (`spec/policy.md`)
- Alignment with industry frameworks (CSA ATF, OWASP ASI, ISO 42001, NIST AI RMF)
- Enterprise readiness requirements

## Meetings

- **Cadence**: Monthly
- **Notes**: Published in [`community/meetings/`](../../meetings/)
- **Agenda**: Posted 24 hours in advance

## Responsibilities

- Review security implications of RFCs and spec changes
- Maintain autonomy level definitions and demotion triggers
- Ensure alignment with CSA AI Trust Framework and OWASP ASI
- Advise on enterprise readiness and compliance requirements
- Review secret handling and configuration security patterns
